# Plan: Distinguish Transport Errors in GetServerOption

## Phases

### Phase 1: Surface transport errors through GetServerOption discriminator
status: approved
approved_at: 2026-05-13

**Goal**: Introduce `*tmux.CommandError` at the `Commander` layer, wire `RealCommander.Run` / `RunRaw` to wrap their non-nil errors, rewrite `GetServerOption` to discriminate option-absence via stderr pattern-family match (`invalid option:`, `unknown option:`, `ambiguous option:`), and let the previously-dead error branches in `TryGetServerOption` and its daemon consumers (`tick()`, `defaultShutdownFlush()`) become live. Tighten the four contract-violation docstrings, reshape the existing `TestGetServerOption` "option does not exist" case to use `*CommandError`, add transport-error / non-exit-error / discriminator-set / try-wrapper-propagation / RealCommander-wrap tests, remove the documented-gap comment block at `cmd/state_daemon_run_test.go:557-565`, and replace it with the previously-blocked `defaultShutdownFlush` err-branch test (plus tick err-branch coverage per the sweep).

**Why this order**: Single phase. The specification's Implementation Ordering section explicitly states that units (1) `CommandError` type, (2) `RealCommander` wiring, and (3) `GetServerOption` discriminator "must land together" and "cannot be split safely" — splitting them produces an intermediate state where every error is still mapped to `ErrOptionNotFound` (callers depend on this) until the discriminator lands. Docstring tightening (4) and test reshape (5) are explicitly permitted to land alongside (3). The spec recommends "single PR, single commit." Bugfix guidance favors minimal scope; there is one root cause (conflation in `GetServerOption`) and one contained fix surface. No checkpoint between units would be independently valuable — the system is only contract-faithful after all three land.

**Acceptance**:
- [ ] `*tmux.CommandError` type exists in `internal/tmux` (exported, struct-literal constructable, with `Error()` formatting per spec rules and `Unwrap()` returning the embedded `Err`).
- [ ] `RealCommander.Run` and `RealCommander.RunRaw` return `*CommandError`-wrapped errors on failure, populating `Stderr` from `(*exec.ExitError).Stderr` for exit errors and leaving `Stderr` empty for non-exit errors; original error preserved via `Unwrap()`.
- [ ] `GetServerOption` returns `("", ErrOptionNotFound)` if and only if the error unwraps (via `errors.As`) to a `*CommandError` whose `Stderr` contains a substring from the unexported `optionAbsentStderrPatterns` slice (`invalid option:`, `unknown option:`, `ambiguous option:`); all other errors propagate unchanged.
- [ ] `TryGetServerOption("@some-marker")` returns `("", false, non-nil-err)` for any non-absent failure — the `if err != nil { return "", false, err }` branch is now reachable and live.
- [ ] Daemon consumers (`cmd/state_daemon.go:95-99` `tick()` and `cmd/state_daemon.go:187-201` `defaultShutdownFlush()`) receive non-nil errors for transport failures and execute their existing warn-log + early-return path; no consumer code is modified.
- [ ] The four production docstrings (`internal/tmux/tmux.go` `TryGetServerOption` + `GetServerOption`, `internal/state/markers.go` `RestoringChecker` + `IsRestoringSet`) accurately describe the implemented contract.
- [ ] Reshaped `TestGetServerOption` "option does not exist" uses a `*CommandError` with a matching stderr; new tests assert: transport-error propagation (`TestGetServerOption_TransportError` parametrised over socket-connect and server-crash stderr), non-exit-error propagation (`TestGetServerOption_NonExitErrorPropagates`), `TryGetServerOption` transport-error propagation (`TestTryGetServerOption_PropagatesTransportError`), discriminator-set coverage (each entry in `optionAbsentStderrPatterns` matches + an unrelated stderr does not), `RealCommander` wrap for exit errors (`TestRealCommander_RunWrapsExitError`), and `RealCommander` wrap for non-exit errors (`TestRealCommander_RunWrapsNonExitError`).
- [ ] The documented-gap comment block at `cmd/state_daemon_run_test.go:557-565` is removed and replaced with the previously-blocked `defaultShutdownFlush` err-branch test (asserts `nil` return + zero commit calls under injected `*CommandError{Stderr: "lost server"}`); `tick()` err-handling branch is covered (new test or updated existing mock per the test-code sweep).
- [ ] `go test ./...` passes; pre-existing tests show no behavioural change in the happy path; `internal/state/markers_test.go` `TestIsRestoringSet :: propagates underlying tmux error` continues to pass unchanged.
- [ ] Pre-implementation sweeps (production-code grep and test-code grep per the spec's "Pre-implementation sweep" section) confirm no caller outside the audited sites depends on the old conflation; any newly-surfaced caller is audited before merge.
