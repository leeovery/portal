- FINDING: Projects page still binds the vim / uppercase / page-jump keys §12.2 mandates dropping
  SEVERITY: high
  FILES: internal/tui/model.go:1080-1093 (newProjectList), internal/tui/model.go:1056-1063 (pinArrowOnlyNav), internal/tui/model.go:2363-2364 (updateProjectsPage delegate)
  DESCRIPTION: §12.2 is unambiguous: "Drop all vim aliases (h/j/k/l, g/G) and PgUp/PgDn/Home/End — move is ↑/↓, page is Ctrl+↑/↓" and "No uppercase bindings anywhere." This applies to both list pages (§12.1 lists the same nav revision for Sessions AND Projects). The Sessions list enforces it: newSessionList calls pinArrowOnlyNav, which rebinds CursorUp/Down to up/down, PrevPage/NextPage to ctrl+up/ctrl+down, and empties GoToStart/GoToEnd — so the banned keys never reach the list's own Update. newProjectList does NOT call pinArrowOnlyNav (it is the only difference between the two constructors), so the Projects list retains the bubbles/list v2 DefaultKeyMap, which binds CursorUp=[up,k], CursorDown=[down,j], PrevPage=[left,h,pgup,b,u], NextPage=[right,l,pgdown,f,d], GoToStart=[home,g], GoToEnd=[end,G] (verified in charm.land/bubbles/v2@v2.1.0/list/keys.go:34-60). updateProjectsPage explicitly intercepts ?, esc, q, x, n, d, e, enter before delegating to m.projectList.Update, but every UN-handled key — j, k, h, l, g, G, pgup, pgdown, home, end, b, u, f — falls through and moves the cursor/page on the Projects page. So a user pressing j/k/h/l/g/G or PgUp/PgDn/Home/End on Projects gets live navigation that §12.2 forbids; the uppercase G binding additionally violates "No uppercase bindings anywhere." This is a genuine behaviour drift, not cosmetic. The existing test that appears to cover this (projects_keymap_test.go:133 "it has no uppercase or vim-alias key in the descriptor") tests the WRONG layer: the keymap descriptor only drives the footer + help-modal DISPLAY, not the live bubbles/list dispatch — so it passes while the actual dispatch leaks the banned keys, giving false assurance.
  RECOMMENDATION: Call pinArrowOnlyNav(&l.KeyMap) in newProjectList (model.go:1080), mirroring newSessionList. Add a dispatch-layer (not descriptor-layer) test asserting that j/k/h/l/g/G/pgup/pgdown/home/end are no-ops on the Projects page cursor — the same coverage the session list has.
