# Drop invalid `-A` flag from `tmux attach-session` argv

v0.5.1 introduced an `-A` flag on the `tmux attach-session` argv at `cmd/open.go:101` (commit `d93fa942`, task `enter-attaches-from-preview-1-2`). tmux's `attach-session` does not accept `-A` — that flag belongs to `new-session`. tmux 3.6a's `list-commands` confirms the only valid flags on `attach-session` are `-dErx -c <dir> -f <flags> -t <target>`. Every outside-tmux attach from the Sessions list now exits immediately with `command attach-session: unknown flag -A`, which is tmux's own error format. Portal is fully broken for bare-shell users on v0.5.1 and v0.5.2; inside-tmux usage is unaffected because that path goes through `switch-client`, and `portal open <path>` is unaffected because that path goes through `new-session -A` where `-A` is valid.

The fix is to drop `-A` from the argv at `cmd/open.go:101` while keeping the `=` exact-match prefix, restoring the pre-v0.5.1 shape but retaining the prefix-collision protection added by the same task. The docstring at `cmd/open.go:79-83` justifies `-A` as "atomic create-or-attach … residual fallback for the TOCTOU window" — that claim is false and must be removed. The unit-test argv expectation at `cmd/open_test.go:1122` (`["tmux", "attach-session", "-A", "-t", "=foo"]`) needs updating to match.

The defect originated in the specification itself: `.workflows/enter-attaches-from-preview/specification/enter-attaches-from-preview/specification.md` §88 and §166 both assert `tmux attach-session -A -t '=<session>'` as canonical argv, and §166 even attributes fictional "auto-create on absent session" behaviour to `-A`. Both spec lines should be corrected as part of the fix so future readers (and any re-derivation from spec) don't reintroduce the same bug.
