# `newSymptomFixture` subprocess spawns full bootstrap against the developer's real `~/.config/portal/hooks.json`, wiping it on every test run

`cmd/state_commit_now_symptom_integration_test.go` defines `newSymptomFixture` (line 337) which is consumed by three default-lane integration tests in the `cmd_test` package: `TestCommitNowSymptom*` (the symptom file itself), `TestCommitNowReentrancy*` (`state_commit_now_reentrancy_integration_test.go`), and `TestCommitNowDaemonMergeStability` (`state_commit_now_daemon_merge_integration_test.go`). Each of these wipes the developer's real `~/.config/portal/hooks.json` to `{}` every time it runs, with no audit trail visible in the developer's `portal.log`.

Verified deterministically on 2026-06-04: `go test -count=1 ./cmd/` (default lane, no `-tags integration`) wiped a 3-entry `hooks.json` to 0 entries in a 20-second run. Bisection by individual test identifier pinned the first observable wipe to `TestCommitNowDaemonMergeStability` (it is the first test alphabetically that uses the fixture; subsequent fixture-using tests then re-wipe the already-empty file and the hash no longer changes).

Mechanism: the fixture sets `t.Setenv("PORTAL_STATE_DIR", stateDir)` (line 358) where `stateDir = t.TempDir()`, then calls `runPortalList(t, binary, fixture)` (line 387) to spawn `portal list` as a subprocess. `portal list` triggers `PersistentPreRunE` → the full eleven-step bootstrap orchestrator. Bootstrap step 11 (`CleanStale`) calls `runHookStaleCleanup`, which resolves the hooks file via `loadHookStore` → `configFilePath("PORTAL_HOOKS_FILE", "hooks.json")`. The subprocess inherits `PORTAL_STATE_DIR` (isolated) but `PORTAL_HOOKS_FILE` is never set, so the helper falls through to `$XDG_CONFIG_HOME/portal/hooks.json` / `~/.config/portal/hooks.json` — the developer's real file. `ListAllPanes` returns the test tmux server's pane set (`_anchor`, `A`, `B`, `_portal-saver`). The hazard guard does not fire (livePanes is non-empty). Every developer hook key is classified as "not in live pane set" and reaped. The store rewrites the file to `{}`.

The asymmetric isolation — `PORTAL_STATE_DIR` isolated, `PORTAL_HOOKS_FILE`/`PORTAL_PROJECTS_FILE`/`PORTAL_ALIASES_FILE` not — creates a blind spot: the subprocess's `hooks: clean-stale entries=N via=internal` audit line lands in the tempdir state directory's `portal.log` (because the logger correctly uses the isolated state dir) and is thrown away with the tempdir at test end, while the destruction lands on the developer's real `hooks.json`. The audit trail and the destruction site point at different filesystems — by design, but unintentionally so. That is why every wipe observed in the past two days has been invisible to `grep hooks: ~/.config/portal/state/portal.log.*`.

This is the same class as the `cleanup-purge-test-no-state-isolation` defect fixed today in commit `ad948ae0` (`cmd/state_test.go` `TestStateUserFacingSubcommandsExitZero`'s `--purge` subtest), but a different target (hooks file instead of state directory) and a different mechanism (subprocess env asymmetry instead of in-process production-cobra-command invocation). The two together explain everything observed in this week's hooks-vanish and log-vanish episodes.

Likely scope of the fix: extend `newSymptomFixture` (and any sibling fixture that spawns a bootstrap-triggering portal subprocess — `runPortalList`, `runPortalCommitNow`, equivalents elsewhere) to isolate `PORTAL_HOOKS_FILE`, `PORTAL_PROJECTS_FILE`, and `PORTAL_ALIASES_FILE` alongside `PORTAL_STATE_DIR`. The canonical helper `portaltest.IsolateStateForTest(t) (env []string, stateDir string)` already returns a complete env covering all four — migrating the fixture to that helper plus `cmd.Env = env` on every subprocess closes the gap deterministically. A guard test that fails whenever a test subprocess inherits unset `PORTAL_*_FILE` envs would prevent future instances of this class.

Severity is high. Silent destruction of user-managed configuration, recurring on every `go test ./cmd/` invocation (which Claude-driven implementation work runs many times per day), no audit-trail visibility, no in-band warning. The fix is mechanical and low-risk; the discovery cost was the bulk of the work.

Relevant files: `cmd/state_commit_now_symptom_integration_test.go` (the fixture, lines 337–411), `cmd/state_commit_now_reentrancy_integration_test.go` and `cmd/state_commit_now_daemon_merge_integration_test.go` (consumers), `cmd/hooks.go:136` (`configFilePath("PORTAL_HOOKS_FILE", "hooks.json")`), `cmd/run_hook_stale_cleanup.go` (the destructive `CleanStale` path that fires under bootstrap step 11), `internal/portaltest/isolated_env.go` (the canonical helper that already covers all four env vars), and `cmd/state_test.go` commit `ad948ae0` (the structural-precedent fix for the sibling defect).
