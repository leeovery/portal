  DESCRIPTION: Registration converges `session-closed` onto `commitNowCommand`, whose only Portal fingerprint is `portal state commit-now` (its body is `run-shell "command -v portal ... && portal state commit-now"` — it does NOT contain `portal state notify`). The teardown path's `portalCommandSubstrings` set is `{portal state notify, portal state signal-hydrate, portal state migrate-rename}` — it omits `portal state commit-now`. Consequence: `UnregisterPortalHooks` / `portal hooks reset` reads `session-closed` per-event, finds the converged commit-now entry, classifies it as non-Portal (no substring matches), and leaves it installed. A Portal-authored hook on a managed event survives teardown, directly contradicting Acceptance Criterion 5 ("removes all Portal entries at any depth on every managed event"). This is a seam defect between the two halves the plan unified. The work derived `portalEvents` from `managedEvents` so registration and teardown provably scan the identical event-set, and `TestPortalManagedEventSetParity` guards it — but that parity is over *events*, not over *fingerprints*. Registration's per-event fingerprint set (which carries `commitNowSubstring` for session-closed) and teardown's flat `portalCommandSubstrings` diverge in a way that makes one registered category unreachable by teardown. The real-tmux teardown-at-depth test (hooks_register_realtmux_test.go) asserts post-teardown that zero entries match `portal state commit-now`, but it only ever seeds `notify` bodies on the two blind events and never seeds a commit-now body on session-closed, so that assertion is vacuously green and the gap is uncovered. The migrate-rename omission from registration is intentional and spec-sanctioned (registration shouldn't install/converge it); the commit-now omission from teardown is the inverse mistake — teardown should reap everything registration installs. The commit-now registration predates this plan (killed-session-resurrects fix), so the latent gap is partly inherited, but this plan rewrote teardown and re-grounded the event-set on `managedEvents` without closing the corresponding fingerprint gap.
  RECOMMENDATION: Add `commitNowSubstring` ("portal state commit-now") to teardown's `portalCommandSubstrings` so `UnregisterPortalHooks` reaps the converged session-closed entry, restoring AC #5. To prevent future fingerprint drift the way `managedEventNames()` prevents event-set drift, consider deriving teardown's fingerprint set from the union of every `managedEvents` entry's fingerprints (plus the legacy migrate-rename substring teardown intentionally retains), so a future hook category added to `managedEvents` automatically widens teardown coverage. Then extend the real-tmux teardown-at-depth test to seed a stacked `commitNowCommand` on session-closed and assert it reaps to zero, so the assertion stops being vacuous.
