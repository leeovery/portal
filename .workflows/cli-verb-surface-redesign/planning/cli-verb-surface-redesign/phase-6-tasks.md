### Task 6-3: Session-name completion (shared helper) for `open` positional, `open -s`, `kill` + exempt completion from bootstrap

**Problem**: Session-name TAB completion is genuinely new — there is no `ValidArgsFunction` in production today. And there is a load-bearing hazard: cobra runs the root `PersistentPreRunE` for the `__complete` command itself, so wiring completion naively would fire Portal's full 10-step bootstrap (start the tmux server + restore sessions) on every TAB press. The completion machinery only skips the *target* command's `execute()`, not the root persistent pre-run chain that precedes the `__complete` command's own `Run`.

**Solution**: (a) Add `__complete` to `skipTmuxCheck` so the completion request returns from `PersistentPreRunE` before any bootstrap. (b) Add a shared session-name completer that builds its OWN `tmux.DefaultClient()` — because the exempt path injects no client into `cmd.Context()`, calling `tmuxClient(cmd)` would panic — sources names from the user-visible (leading-`_`-filtered) `ListSessionNames` view, prefix-filters, and returns `ShellCompDirectiveNoFileComp`. (c) Wire it to `open`'s bare positional, `open --session`, and `kill`'s positional.

**Outcome**: `portal open <TAB>` / `portal open -s <TAB>` / `portal kill <TAB>` suggest user-visible session names, with zero bootstrap side-effects on a TAB press and graceful empty results when the server is down.

**Do**:
- In `cmd/root.go`, add `"__complete": true` to the `skipTmuxCheck` map. This covers the `__completeNoDesc` alias too: cobra derives `Name()` from `Use` (`"__complete [command-line]"`), so `Name()` is always `"__complete"` regardless of which alias the shell used; the `PersistentPreRunE` skip loop starts at the executed command (`cmd.Name() == "__complete"`) and short-circuits.
- New file `cmd/completion.go`:
  - `var completionSessionNames = func() []string { names, err := tmux.DefaultClient().ListSessionNames(); if err != nil { return nil }; return names }` — the injectable production seam; sourced ONLY from `ListSessionNames` (never `has-session` or a raw `list-sessions`) so the leading-`_` filter in `Client.ListSessions` applies and internal `_portal-saver`/`_portal-bootstrap` are never suggested.
  - `func completeSessionNames(toComplete string) ([]string, cobra.ShellCompDirective)` — call the seam, keep entries where `strings.HasPrefix(name, toComplete)` (cobra does NOT prefix-filter a `ValidArgsFunction`'s returns — verified in v1.10.2 `getCompletions`, so the helper must filter), and return `cobra.ShellCompDirectiveNoFileComp`.
  - The completer must NEVER call `tmuxClient(cmd)` (it panics without `PersistentPreRunE`).
- Wire in `cmd/open.go` `init()`: `openCmd.ValidArgsFunction = func(cmd *cobra.Command, args []string, toComplete string) ([]string, cobra.ShellCompDirective) { return completeSessionNames(toComplete) }` and `_ = openCmd.RegisterFlagCompletionFunc("session", func(cmd *cobra.Command, args []string, toComplete string) ([]string, cobra.ShellCompDirective) { return completeSessionNames(toComplete) })` (the `--session` flag is added in Phase 2).
- Wire in `cmd/kill.go` `init()`: `killCmd.ValidArgsFunction = func(cmd *cobra.Command, args []string, toComplete string) ([]string, cobra.ShellCompDirective) { if len(args) > 0 { return nil, cobra.ShellCompDirectiveNoFileComp }; return completeSessionNames(toComplete) }` (kill is `ExactArgs(1)`, so only the first positional completes).

**Acceptance Criteria**:
- [ ] `skipTmuxCheck["__complete"] == true`; executing argv `["__complete","open",""]` does NOT invoke the bootstrap orchestrator (recordingRunner Run count 0).
