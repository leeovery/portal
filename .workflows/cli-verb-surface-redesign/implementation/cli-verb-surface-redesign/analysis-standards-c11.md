A full fresh pass over the current working-tree surface (cmd/open*.go, root.go, doctor.go, uninstall.go, kill.go, hooks.go, completion.go, state*.go, resolver/*, spawn/*, log/*, tmux hooks_count/unregister, main.go) found no spec drift and no project-skill MUST-DO/MUST-NOT-DO violations.

Verified conformant against the governing decisions:
- Public verb collapse: attach/spawn/clean/state-status/state-cleanup source files deleted; open is the single public session verb; no lingering "portal attach|spawn|clean" references in production help/init.
- Target resolution: glob pre-check (HasGlobMeta *?[) is session-domain-by-construction; precedence chain exact-session → path → alias → zoxide; session-domain resolution bound to the user-visible ListSessionNames view (internal _portal-* sessions never matchable).
- Miss handling: total miss is a hard fail (singleMissError "nothing resolved for '%s' — try -f %s"; aggregatedMissError with no -f for N>=2); no TUI-picker fallback anywhere.
- Pinned-domain hard-fail contract: all four pins (-s/-p/-z/-a) resolve one domain, never mint-to-picker, hard-fail on miss; -z surfaces ErrZoxideNotInstalled verbatim while the bare chain swallows it; -a reaches shadowed keys; -p stats the literal path (glob-named dir reachable). Pins emit no resolve line.
- Closed log taxonomy: single log.For("resolve") binding in cmd/open.go; emit is Info("resolved", target/domain/resolved_path) — exactly the spec-governed attr set, glob/pin-gated via the shared emitResolveDecision. spawn-component emitters (logemit.go) use only the closed keys (session/ack/detail/resolution/terminal/bundle_id/opened/total/batch). uninstall reuses the daemon component and doctor reuses the bootstrap component — no new component or attr-key invented at any call site.
- Command passthrough: -e/-- mutually exclusive; empty -e rejected; command is mint-scoped (attach-target command → NewUsageError in both single openResolved and multi runOpenBurstWithDeps zero-mint guard); no word-splitting (composeOpenArgv appends command tokens byte-identically after --; -e stays one unit); mint-only command with no target opens the picker (threaded to openTUI).
- Multi-target burst: os.Args-ordered target recovery (spec-endorsed), net-N (SplitTriggerFirst, trigger connects LAST), atomic read-only pre-flight, leave-what-opened, per-window ack, unsupported/NULL terminal blocks N>=2. --ack hidden (MarkHidden), best-effort write (writeAckMarker logs DEBUG on failure then connects).
- doctor: bootstrap-exempt; read-only catalog + --fix (prune stale hooks with down-server mass-deletion hazard guard, prune stale projects filesystem-only, unconditional log-sweep outside exit-code); host-terminal line is checkInfo (never drives exit); down server → checkFail → non-zero; exit 0 iff all pass; --fix re-diagnoses and drives exit solely from post-repair results.
- uninstall: bootstrap-exempt; runtime-only (kill _portal-saver then unregister hooks, killSaver before unregister); touches no files; idempotent no-op when server down; byte-exact two-line completion message on every path including partial failure; leaves all sessions incl. _portal-bootstrap.
- kill: single + exact (cobra.ExactArgs(1)), no globs, no resolution chain; unchanged.
- hooks → hook: canonical "hook" with permanent silent Aliases ["hooks"] (no cmd.Deprecated); skipTmuxCheck keys on c.Name()=="hook".
- state namespace fully hidden: parent + all six children (daemon/hydrate/notify/signal-hydrate/migrate-rename/commit-now) individually Hidden:true; prefix strings preserved for tmux idempotency matchers.
- Bootstrap exemption: skipTmuxCheck adds doctor + uninstall, keeps state, drops clean; bare portal has no Run/RunE (prints help, never picker).
- Daemon automation: stale-project prune folded into the daemon (maybeRunProjectCleanup, ~hourly throttle) alongside the ~10s hook cleanup — matches the spec's "folds into the daemon's automation."
- Tab completion: bare positional + -s + kill complete session names (user-visible ListSessionNames); -a completes alias keys; -p/-z left unregistered (shell/zoxide default); hidden --ack and hidden state auto-excluded.
- Exit codes: usage errors → *cmd.UsageError → code 2; doctor-unhealthy silent-exit sentinel → code 1 (report already on stdout); everything else code 1. Root sets SilenceUsage/SilenceErrors.
- Retained roleClean process-role arm and the "clean" log component (bootstrap FIFO/marker sweep) are correctly kept: the redesign amended the closed taxonomy only for "resolve", so removing them would need a taxonomy amendment out of scope here — documented as intentional, not drift.

House-style capitalised user-facing error strings ("No session found", "No alias found", "No zoxide match", "Directory not found") carry explicit //nolint:staticcheck directives and are the deliberate byte-compat choice with the retired attach path — an intentional project-convention override of the generic lowercase-error rule, not a violation.
