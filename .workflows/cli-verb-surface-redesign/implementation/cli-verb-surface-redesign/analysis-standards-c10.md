Verification notes (what was checked and confirmed, not defects):

- Command surface (spec § Command Surface Summary). rootCmd registers exactly the specified public/hidden set: open, kill, list, alias, hook, doctor, uninstall, init, version, state (Hidden), plus cobra's completion. attach.go / spawn.go / clean.go / state_status.go / state_cleanup.go are deleted outright (no aliases, no deprecation shims) — confirmed against git history and cmd/retired_surface_test.go. internal/state/status.go likewise deleted.

- open resolution grammar (spec §§ Target resolution precedence, Glob targets, Miss handling). Precedence chain exact-session → path → alias → zoxide is single-sourced in resolver.QueryResolver; glob pre-check routes glob-bearing targets to the burst (session-domain by construction); total miss hard-fails with the byte-exact "nothing resolved for '<t>' — try -f <t>" (singleMissError) and the aggregated no-`-f` multi form. Session-domain resolution matches only the user-visible (leading-underscore-filtered) ListSessionNames view, so open/-s/glob can never reach _portal-*.

- Pinned-domain hard-fail contract (spec § Pinned-domain contract; § attach — Retired). All four pins (-s/-p/-z/-a) resolve one domain and hard-fail on a miss with no picker fallback: single-target via resolvePinAndOpen → resolve(...) error propagation (ResolveSessionPin/ResolvePathPin/ResolveAliasPin/ResolveZoxidePin never return a MissResult); multi-target via collected misses → aggregated abort. -z surfaces ErrZoxideNotInstalled verbatim while the bare chain swallows it. The pin dispatch loop sits before the no-target picker early-return, so `open --session x` never pops a TUI. isTUIPath/anyOpenDomainPin keep pinned invocations on the synchronous direct path.

- Command passthrough mint-scoping (spec § Command passthrough). -e/-- mutual-exclusion enforced (parseCommandArgs); command guarded to mint outcomes only (openResolved *SessionResult arm + runOpenBurstWithDeps zero-mint guard, both via the single commandAttachOnlyMessage); zero-mint-with-command → usage error; command-parity (no word-splitting) preserved end-to-end (composeOpenArgv carries the command slice byte-identically to spawned mint windows and to the trigger's LocalMint). Mint-only command with no target opens the Projects-mode picker.

- Hidden --ack receipt (spec § Hidden --ack flag). Renamed from --spawn-ack, MarkHidden'd, validated at top of RunE (usage error on malformed), best-effort write as the last act before handoff (writeAckMarker), false-negative-safe. Spawned windows exec `open --session|--path ... --ack <batch>:<token>` (composeOpenArgv) — no retired verb.

- doctor / uninstall / hook (spec §§ doctor, uninstall, Remaining Verbs). doctor implements the full 7-item catalog with the correct exit-code contract (0 iff all pass; down server → fail via daemon/saver/hooks; host-terminal line checkInfo, never drives exit; --fix re-diagnoses and exit is post-repair only; log-sweep unconditional and outside the loop; down-server stale-hook hazard guard defers pruning). uninstall is runtime-only, touches no files, byte-exact two-line message on every path, idempotent no-op when server down, kill-saver-before-unregister ordering. hook is canonical with the permanent silent `hooks` cobra alias (plain Aliases, no Deprecated). All three bootstrap-exempt in skipTmuxCheck.

- Closed log taxonomy (spec § Wrong-guess feedback; CLAUDE.md log section). Exactly 17 components in use; the new `resolve` component is bound only in cmd/open.go and emits `resolved` with the locked attrs target/domain/resolved_path (emitResolveDecision, gated on !HasGlobMeta, emitted on miss too, suppressed for pins/globs). internal/resolver emits no log records (its internal/log import is only the CombinedOutputWithContext exec helper). spawn-component attr keys at call sites stay within the spec-governed set (batch/detail/session/ack/resolution/terminal/bundle_id/opened/total). No call-site component or attr invention. The `clean` log component is retained legitimately for subsystem sweeps (bootstrap / fifo_sweep), distinct from the deleted `clean` command.

- Tab completion (spec § Tab Completion). open positional + -s → session names; -a → alias keys; -p/-z left unregistered (shell default); kill positional → session names; all sourced from the user-visible / Portal-owned namespaces and safe on the bootstrap-exempt __complete path.

- Conventions. Root sets SilenceUsage/SilenceErrors; all commands use RunE + cobra Args validators (ExactArgs/NoArgs/ArbitraryArgs), cmd.OutOrStdout()/ErrOrStderr(); os.Exit owned solely by main with UsageError→2 / general→1 mapping; error strings lowercase except the deliberately-preserved capitalised user-facing "No session/alias found" messages (nolint-annotated per house style). Bare `portal` prints help (no Run/RunE on root), not the picker; x/xctl map to `portal open`.

Note (informational, not a defect / no change recommended): the spec deletes the `clean` command, leaving log.ResolveProcessRole's `case "clean"`/roleClean arm unreachable.
