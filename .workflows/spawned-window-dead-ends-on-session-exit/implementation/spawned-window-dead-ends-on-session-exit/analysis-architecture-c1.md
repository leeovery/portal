  DESCRIPTION: The whole fix is a quote-nesting transformation (inner per-element `'\''` quoting → re-quoted for the `bash -lc` payload → AppleScript-escaped). Acceptance criterion #7 explicitly asks to "assert the wrapper shape against the correctly-escaped expected string (the `'\''`-escaped nesting, not the schematic form)". The delivered tests never do this. Every `want` is recomputed with the same production function it is meant to guard: `TestWrapWithShellFallback` asserts `wrapped[2] == renderCommandString(cmd) + shellFallbackSuffix` (tautological — a bug in `renderCommandString`'s escaping is reproduced on both sides), and `TestGhosttyEmbed`'s round-trips compare against `wrapWithShellFallback(cmd)` after decoding with `decodeRenderedArgv`, a ~47-line hand-rolled POSIX-shell-quote parser (lines 55-101) that inverts `shellQuote`/`renderCommandString`. That decoder is itself untested and must stay in lockstep with the production quoting strategy; a symmetric encode/decode bug passes the round-trip falsely. The only independent literal anchors are two substring fragments — the `command:"'bash' '-lc' ` prefix and the `'\\''` doubled-escape signature — which pin fragments, not the full nesting. Net: the guard is weaker than the spec prescribes at exactly the seam most likely to regress (the nested single-quote escaping this fix exists to get right).
  RECOMMENDATION: Add at least one assertion that pins the full correctly-escaped expected string as a hand-written literal (the `'\''`-escaped, AppleScript-escaped `command:"..."` body) for the canonical attach argv and for the quote-sensitive mint fixture — per acceptance criterion #7. Keep the round-trip decoder as a supplementary check, but let the golden literal, not `decodeRenderedArgv`/re-called production functions, be the primary oracle so a subtle change in `renderCommandString`'s escaping is actually caught rather than mirrored.
SUMMARY: Production change is architecturally sound — correctly scoped to the Ghostty adapter (not shared composition), a clean pure argv-agnostic helper, no API-surface change, no dangling flag references. The one concern is test-seam guard strength: the command-composition tests validate self-consistency (recomputed expectations + a parallel unverified decoder) instead of pinning the escaped golden string acceptance criterion #7 requires, leaving a symmetric-bug blind spot at the fix's central quote-nesting seam.
